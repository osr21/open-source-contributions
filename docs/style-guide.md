# Style Guide

## General Principles
- Clarity over cleverness
- Consistency across the codebase
- Self-documenting code

## Naming Conventions
- Variables: camelCase
- Constants: UPPER_SNAKE_CASE
- Files: kebab-case
- Classes: PascalCase

## Comments
- Explain WHY, not WHAT
- Keep comments up to date with code changes
- Use JSDoc for public APIs
