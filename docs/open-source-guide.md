# Beginner's Guide to Open Source Contributing

## Why Contribute?
- Learn from real-world codebases
- Build your public portfolio
- Meet other developers
- Give back to tools you use every day

## Finding Your First Project
1. Look at projects you already use
2. Search GitHub with labels: `good first issue`, `help wanted`, `beginner-friendly`
3. Browse [goodfirstissue.dev](https://goodfirstissue.dev) or [up-for-grabs.net](https://up-for-grabs.net)

## Before You Start
- Read the CONTRIBUTING.md
- Check if an issue exists before opening a new one
- Comment on the issue to say you're working on it

## Making Your First Contribution
1. Fork the repository
2. Clone your fork locally
3. Create a focused branch
4. Make small, focused changes
5. Write or update tests
6. Submit a pull request with a clear description

## Tips for a Great PR
- One feature or fix per PR
- Reference the issue it closes
- Be responsive to review feedback
- Keep commits clean and descriptive

## Don't Be Discouraged
Everyone starts somewhere. Maintainers appreciate all contributions — even small documentation fixes make a difference.
